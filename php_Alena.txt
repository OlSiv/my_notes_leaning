во всех массивах есть ключ (слева) и значение (справа). Просто в неассоциативных массивах ключом являются числа, а в ассоциативных строки. Ведь даже $array = [1,2,3,4]; имеет ключи 0,1,2,3, хотя их явно «не видно»

массив - инструмент для хранения и обработки коллекций(множеств) данных 

массив может содержать различные типы данных 

--
<?php 

$array = array(1,2,3,4); // - использовалось до версии 5.4

echo $array;   // - не выведет, будет ошибка 
print_r($array);

$array = [1, 2, 3]  // - после версии 5.4 

$arr2 = ['a', 'b', 'c'];
print_r($arr2);
----

типы массивов - индексированные массивы и ассоциативные 

индексированные - ключи представлены в виде чисел (0, 1, 2, 3 ...)
ассоциативные - ключи представлены строковыми значениями -
--
<?php

$array3 = [
    'first' => 1,
    'second' => 2,
    3, 4
];

print_r($array3);

$array4 = [
    'first' => 1,
	3,
    'second' => 2,
    4
];

print_r($array4);

----
Array ( [first] => 1 [second] => 2 [0] => 3 [1] => 4 )
Array ( [first] => 1 [0] => 3 [second] => 2 [1] => 4 )
------

еще есть многомерные массивы - это массивы которые содержат другие массивы, но их мы рассмотрим в отдельном уроке

если добавить в массив элемент с уже существующим ключом, то элемент с этим ключем в массиве исчезнет, так как будет заменен новой парой ключ-значение -

цикличный оператор FOREACH предназначен для итерации (перебору) по элементам массива (или объекта)

--
<?php 

$arr = [1, 2, 3, 'a', 'b', 'c', 'name' => 'Alex', 'age' => 42];

    print_r($i . PHP_EOL);

интерпретатор идет по коду сверху вниз 

прервать процесс после первого элемента - 
--
<?php 

$arr = [1, 2, 3, 'a', 'b', 'c', 'name' => 'Alex', 'age' => 42];

foreach ($arr as $i) {
    print_r($i . PHP_EOL);
	die();
}
----
1 
------
- die() - полностью завершает выполнение скрипта php 

break в действии в цикле foreach - когда break встречается внутри цикла - выполнение цикла останавливается и управление передается на следующую строку кода -
--
<?php 

$arr = [1, 2, 3, 'a', 'b', 'c', 'name' => 'Alex', 'age' => 42];

foreach ($arr as $i) {
    print_r($i . PHP_EOL);
	break;
}
----
1
------

continue - пропустить, продолжить выполнение цикла -
--
<?php 

$arr = [1, 2, 3, 'a', 'b', 'c', 'name' => 'Alex', 'age' => 42];

foreach ($arr as $i) {
    if ($i === 2 or $i === 'c') {
        continue;
    }
    print_r($i . PHP_EOL);
}
----
- выведет все значения кроме 2 и 'c'

можно работать не только со значениями массивов но и с ключами - 
--
<?php 

$arr = [1, 2, 3, 'a', 'b', 'c', 'name' => 'Alex', 'age' => 42];

foreach ($arr as $k => $v) {
    print_r($k . '=' . $v . PHP_EOL);
}
----

2=3
3=a
4=b
5=c

создадим новый проект - File - New Project - oop
поставим галочку - Don't ask again - нажать New Window

в корне проекта создать директорию - app (там обычно содержатся файлы с классами и другие служебные файлы для работы с приложениями, тут мы и будем создавать наши классы)

в корне проекта создать директорию - public (тут находятся файлы, которые доступны напрямую через web, то есть это корень нашего web-сервера, откуда запускаются страницы и обрабатываются http-запросы) - в этой папке создать стартовый файл - index.php (это главный входной файл приложения, отвечает за обработку запросов, подключение других файлов, инициализацию объектов) - то есть классы будут в app, а в index.php мы будем создавать объекты этих классов 

класс - это шаблон, чертеж для создания объекта 

в классе содержаться свойства объекта(переменные) и методы(функции) определяющие поведение объекта 

объект - это экземпляр класса 

в папке app - New - PHP class 
Name - Student

есть модификаторы доступа - public, protected, private 

--
app/Student.php
--
<?php

namespace App;

class Student
{
    public $name;
    public $age;
  public function great() {
      return 'Hello! Im student!';
  }
}
----

в index.php мы создаем и инициализируем объект 

--
public/index.php 
--
<?php 

$student1 = new Student();

print_r($student1);
----

- $owner (владелец счета)
- $balance (текущий баланс счета)
- $accountNumber (номер счета)
- $currency (валюта счета)
2 - 
создайте методы класса BankAccount:
- deposit($amount) - метод для внесения средств на счет 
- withdraw($amount) - метод для снятия средств со счета, уменьшает баланс на указанную сумму, если на счете достаточно средств 
- getBalance() - метод для получения текущего баланса счета 
- displayAccountInfo() - метод о выводе информации о счете (владелец, номер счета, текущий баланс и валюта)

--
app/BankAccount.php
--
<?php

namespace app;

class BankAccount
{
  public $owner;
    public $balance;
    public $accountNumber;
    public $currency;
  
  public function deposit($amount) {
      $balance += $amount;
  }
  
  public function withdraw($amount) {
      if ($balance >= $amount) {
        $balance -= $amount;
    }
  }
  
  public function getBalance() {
      return $balance;
  }
  
    public function getBalance() {
      print_r($owner);
      print_r($balance);
      print_r($accountNumber);
      print_r($currency);
  }
}
----

#3 ООП в PHP: подключаем Автозагрузку классов | Серия уроков для получения оффера
https://youtu.be/hXLJcCWrxmM

Полезные ссылки:
https://ru.hexlet.io/courses/php-setup-environment/lessons/composer-autoload/theory_unit

автозагрузка - что это и для чего она нужна, почему она важна для работы с классами и объектами в ООП 

подключим Student.php к index.php c помощью команды require_once, это команда нужна для указания пути в текущем скрипте -
--
public/index.php 
--
<?php 

use app\Student;
require_once '../app/Student.php';

$student1 = new Student();
print_r($student1);
print_r($student1->great());
----

изучить самостоятельно чем команда require_once отличается от команд -
require
include 
include_once 

но если в папке app будет слишком много классов, то чтобы не прописывать каждый класс вручную можно использовать автозагрузку классов с помощью композера, который генерирует автозагрузчик - файл autoload.php, который автоматически будет загружать классы, когда они используются в нашем коде 

создадим файл composer.json в корне проекта - 
oop - New - File - composer.json

самостоятельно погуглить и изучить формат json 

autoload - указывает какие классы и из каких директорий автоматически загружать при использовании php-скриптов 

psr-4 - стандарт загрузки, который предполагает следующие правила - префикс пространства имен соответствует относительному пути от директории вендора (там где у нас установлен композер) 

--
composer.json
--
{
    "autoload": {
      "psr-4": {
        "App\\": "app/"
      }
      }
    }
----

запустим композер командой через терминал находясь в папке oop -
composer dump-autoload 

- эта команда сгенерирует файлы автозагрузчика на основе прописанных нами правил в файле composer.json 

в результате запуска должна появиться папка vendor в которой будет файл autoload.php 

кроме данной команды есть ещё команды composer install и composer update, но с ними мы познакомимся позже, когда будем устанавливать пакеты и зависимости 

--
app/Student.php
--
<?php

namespace App;

class Student
{
    public $name;
    public $age;
  public function great() {
      return 'Hello! Im student!';
  }
}
----
- в класс Student добавить строку - namespace App; - это пространство имен, способ структурирования классов, интерфейсов, функций и прочего в нашем коде - основные принципы: изоляция кода (позволяет избежать конфликтов имен), можно регулировать где есть какой файл, улучшение читаемости, поддержка автозагрузки 

в файле index.php укажем путь к файлу autoload.php -
--
public/index.php 
--
<?php 

require_once '../vendor/autoload.php';

$student1 = new Student();
print_r($student1);
print_r($student1->great());
----

$student1 = new Student(); - подсвечивается желтым, наведем на него и нажмем Import class 
- после тега <?php появилась строка - use App\Student; -
--
public/index.php 
--
<?php 

use App\Student; 

require_once '../vendor/autoload.php';

$student1 = new Student();
print_r($student1);
print_r($student1->great());
----
