Реляционные базы данных (видео) -
Основной вид баз данных - реляционные базы данных. Это классика. К тому же, именно с ними вам предстоит работать. Посмотрите наш видеоурок по этой теме

реляционные БД - это базы данных, где данные хранятся в виде связанных между собой некими отношениями таблиц (от слова relation - отношение)

Основные составляющие реляционной БД - 
- таблица 
- строка 
- столбец 
- значение 
- первичный ключ - primary key 
- внешний ключ - foreign key 
- связь 
- NULL (отсутствие значения)

первичный ключ не обязательно должен состоять из одного столбца, он может быть и комбинацией нескольких столбцов (составным)

Нормализация БД (видео) -
Нормализация - одно из важнейших понятий при проектировании и работе с базами данных. Многие им пренебрегают, называя нормализацию «сухим сводом правил». Но это не так.

Посмотрите наш видеоурок на эту тему - мы расскажем про первые нормальные формы + поделимся своим опытом. Кстати, ниже в тексте этого урока вы найдете еще несколько примеров на эту тему.

Цель нормализации - исключить дублирование данных и сделать работу с базой данных максимально удобной, чтобы не было задвоений информации, лишних зависимостей и т.д. 

Всего нормальных форм 6-8, но важно помнить 3 нормальные формы - 

нормальная форма 1 - в каждой строке должно быть только одно значение для каждого столбца 

не так -
1 - Вася Петров - Python, SQL

а так - 
1 - Вася Петров - Python
2 - Вася Петров - SQL

нормальная форма 2 - соблюдается нормальная форма 1 + каждый неключевой столбец (не являющейся первичным ключом) неприводимо зависит от каждой части составного ключа 

нормальная форма 3 - соблюдается нормальная форма 2 + нет транзитивных зависимостей между неключевыми признаками 

различие 2 и 3 нормальных форм - во 2-ой нормальной форме зависимость неключегого признака от нескольких частей составного ключа, а в 3-й нормальной форме зависимость неключевых признаков

Нереляционные БД (видео) -
Про реляционные базы данных мы с вами уже поговорили - давайте теперь обсудим другой класс БД, нереляционные. Сейчас они тоже набирают популярность и довольно часто используются на разных проектах в качестве дополнительных под конкретные задачи.

Нереляционные БД хранят данные иначе - 
- пары ключ-значение 
- колоночные СУБД
- графы 

NoSQL актино используют для хранения текста, изображений, слабо структурированной информации и т.д. 

Реляционные БД -
- PostgreSQL
- MySQL
- Oracle 
- MS SQL Server 
- MariaDB

Нереляционные БД -
- MongoDB
- DynamoDB
- HBase 
- Redis 
- Cassandra 

При выборе СУБД следует руководствоваться -
- решаемой задачей 
- операционной системой 
- бюджетом
