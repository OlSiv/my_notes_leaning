************************************************************************

C++ подходит и для высокоуровневой, и для низкоуровневой разработки. Его используют для маленьких проектов, крупных продуктов и даже для создания операционных систем.

На C++ пишут в Яндексе, Amazon, Microsoft, Google, Adobe. Ведь аналогов по производительности у этого языка пока нет.

Отсюда и специфика задач, которые решаются на языке C++. Все они требуют, чтобы код работал быстро:
- высокоэффективный бэкенд,
- сетевые приложения,
- разработка игр,
- разработка оборудования (встраиваемые системы, робототехника, медицинская техника и устройства для умного дома).

Владеть языком C++ — значит знать синтаксис, эффективно использовать встроенные возможности, уметь работать с основными стандартными библиотеками, алгоритмами и структурами данных. А ещё разбираться в ООП и реализовывать его в коде, ведь C++ — объектно-ориентированный язык.

Акцент этого курса — на сам язык программирования и его особенности. После прохождения курса вы будете знать достаточно, чтобы самостоятельно осваивать узконаправленные библиотеки или другие инструменты для разработки на C++.

Звучит непросто, и это на самом деле так. C++ — сложный язык, особенно для новичков. Но этот курс устроен так, чтобы освоить C++ смог каждый. А знание C++ открывает большие возможности для трудоустройства. Главное — посвятить обучению достаточно времени и усилий.

Наш курс состоит из нескольких тематических блоков — модулей. Каждый модуль делится на двухнедельные отрезки. Такой отрезок называется «спринт». Спринты состоят из тем, а темы — из уроков.

В течение одного спринта вы изучаете теорию и выполняете задания в онлайн-тренажёре. Каждый спринт завершается итоговым заданием — самостоятельным проектом.

Проект нужно будет сдавать на проверку код-ревьюеру. Советуем выделить время на правки: иногда исправление кода так же трудоёмко, как написание.

Каждый проект — это задача, с которой разработчик может столкнуться в реальности.

Успешное выполнение проекта зависит от:
- технических знаний и навыков,
- понимания логики построения кода,
- умения запросить информацию и пояснить свой запрос,
- навыков системного мышления,
- способности делиться своими результатами с командой.

Во время обучения нормально уставать, тревожиться и грустить. 

Знакомство с языком программирования начинается с программы, выводящей текст “Hello world”. На C++ это будет так:
--
cout << "Hello world!"s << endl; 
----

Чтобы запустить эту программу, нужно сначала перевести её из человекочитаемого вида в тот, который понятен процессору. То есть преобразовать из исходного кода в двоичный. Делает эту работу специальная программа — компилятор.

В наш тренажёр компилятор уже встроен. Пока вам предстоит работать в нём. А к концу вводной части будет рассказано, как написать полноценную программу.

Работа с тренажёром выглядит так: вы пишете решение в поле ответа, а затем нажимаете кнопку «Запустить» или «Проверить».

«Запустить» — выполнит вашу программу и покажет её результат.
«Проверить» — отправит задачу на проверку. Если решение верное, вам откроется доступ к следующему уроку. Если неверное — вы увидите предположение, в чём ошибка.

Посмотрим на код, который вы только что запускали - Начинается он со слова cout (читается «‎си-аут»‎), которое обозначает стандартный поток вывода. Ведь прежде, чем вывести текст, нужно сообщить, куда его вывести. Мы решили вывести в окно терминала и выбрали cout. При запуске этой программы на компьютере текст появится в окне консоли.

Другим примером места, куда можно вывести информацию, будет файл, канал связи в сети, звуковой канал наушников или другая программа. При этом команды вывода не поменяются. Это — универсальный интерфейс.

Слово cout надо написать маленькими буквами, иначе при компиляции возникнет ошибка. В C++ регистр символов имеет значение.

Данные в поток направляются операцией <<. Первое, что мы отправили в этой маленькой программке, — строка Hello from C++! ^_^. 

Чтобы показать компилятору, что это не код на C++, а текст на понятном нам языке, нужно заключить строку в двойные кавычки. В таком случае компилятор не будет пытаться разобрать команды, а просто аккуратно, символ за символом, прочитает и сохранит всё, что написано внутри кавычек. Дополнительно после кавычек добавлена буква s. Код, задающий строку, включая кавычки и букву s, называется строковым литералом.

Добавлять букву s в конце литерала не обязательно. Так мы подсказываем
компилятору конкретный тип строки, в данном случае string. В основном курсе вы ещё вернётесь к строковым литералам. 

Второе, что мы вывели в поток, — объект endl (читается «эндл»). Он вполне соответствует названию — всё, что выведено в поток после него, будет напечатано с новой строки:
--
cout << "Hello from C++! ^_^"s << endl << "Greetings to all!"s << endl; 
----

Такой код содержит два endl, а значит, на выводе будет уже две строки:
--
Hello from C++! ^_^
Greetings to all! 
----

Кажется, что последний endl не имеет смысла, ведь после него ничего не выведено. Но, тем не менее, завершать строку после каждого вывода — хороший тон.

И последнее, что есть в программе, — точка с запятой. Это обязательный знак, им завершается оператор. В C++ опускать точку с запятой нельзя, недостаточно просто начать писать новую команду с новой строки.
--
cout << "Hello,"s << endl << "C++"s << endl; 
----

Вряд ли настоящая программа будет состоять из одной команды. Для вас хорошие новости — количество команд не ограничено. Реальные программы содержат тысячи, десятки тысяч и даже миллионы строк:
--
// Выводим две строчки
cout << "To be, or not to be,"s << endl;
cout << "that is the question"s << endl; 
----

Код выведет такой текст:
--
To be, or not to be,
that is the question 
----

Программисты называют команды операторами. Но в курсе будем стараться избегать этого термина. Не только потому, что слово «команда»‎ проще, но и потому, что может возникнуть путаница. Совсем скоро вы изучите операции. Они такие же, как в математике: плюс, минус, умножить, равно. Но есть и специфические, встречающиеся только в программировании.

Первая строка в коде выше необычная, она начинается со знака //. Это комментарий — часть кода, предназначенная не для компилятора, а для другого программиста, читающего этот код. В C++ есть два вида комментариев:

Однострочный. Он начинается с // и продолжается до конца строчки:
--
// Выводим две строчки
cout << "To be, or not to be,"s << endl; // первая строчка
cout << "that is the question"s << endl; // вторая строчка
----
   
Многострочный. Он может занимать несколько строчек, заключённых между знаков /* и */:
--
/*
Цитата из трагедии Уильяма Шекспира «‎Гамлет».
Акт III сцена I.
Гамлет, Офелия, Полоний, Король.
*/
cout << "To be, or not to be,"s << endl;
cout << "that is the question"s << endl;
----

Видя комментарий, компилятор понимает, что этот текст не для него и пропускает часть кода.

Забавно, что использование комментария часто не соответствует его виду.
Однострочный комментарий используют для длинных пояснений:
--
// Цитата из трагедии Уильяма Шекспира "Гамлет".
// Акт III сцена I.
// Гамлет, Офелия, Полоний, Король.

cout << "To be, or not to be,"s << endl;
cout << "that is the question"s << endl;
----

А многострочный — для маленького пояснения внутри строки:
--
cout << /* строка */ "To be, or not to be,"s << /* манипулятор */ endl;
cout << /* строка */ "that is the question"s << /* манипулятор */ endl;
----

Если вы просто напишете " внутри строки, компилятор решит, что строка закончилась. Чтобы всё-таки разместить знак кавычки, перед ним ставят символ \. Это действие называется экранированием — мы показываем компилятору, что следующий знак имеет специальное значение:
--
cout << "Quote from \"Hamlet\":"s << endl;
cout << "To be, or not to be,"s << endl;
cout << "that is the question"s << endl; 
----

Результат:
--
Quote from "Hamlet":
To be, or not to be,
that is the question 
----
- Как видно, символ \ бесследно исчез.

Также, если внутри строки написать \, то проблемы возникнут с самим знаком экранирования. Вместо того чтобы поместить его в строку, компилятор будет анализировать следующий символ. Чтобы вставить \ внутрь строки, надо перед ним поставить ещё один символ \:
--
cout << "Используйте символ \\ для экранирования \" и самого \\"s << endl;
----

Результат:
--
Используйте символ \ для экранирования " и самого \ 
----

Каждое экранирование влияет только на следующий символ. Шесть слешей превращаются в три.

\" и \\ — не единственные специальные команды, допустимые внутри строки. Также используются \t для табуляции, \n для перехода на новую строку, \r для перехода в начало текущей строки. Есть и другие — все они перечислены на сайте cppreference.com. Такие команды называются escape-последовательностями. Запоминать их необязательно, в курсе будут использоваться только основные.

Лучшие друзья разработчика — это ошибки

Выполняя предыдущие задания вы, вероятно, уже сталкивались с ними — ошибками компиляции. Это загадочные сообщения на английском языке, которые иногда появляются в окне результата при запуске или проверке программы.

C++ славится запутанными сообщениями об ошибках, но к концу курса вы научитесь понимать их и, конечно, исправлять.

Рассмотрим пример:
--
cout << "Very nice day!"s << endl 
----

Казалось бы, корректный код, но компилятор выдаёт сообщение:
--
/exposed/submission/submission.cpp: In function ‘int main()’:
/exposed/submission/submission.cpp:6:37: error: expected ‘;’ before ‘}’ token
    6 |     cout << "Very nice day!"s << endl
      |                                      ^
      |                                      ;
    7 | }
      | ~                         
----

Суть ошибки указана после слова error: expected ‘;’ before ‘}’ token. Компилятор прямо говорит нам: вставьте знак ;

Только набравшись опыта, вы узнаете, что сообщения об ошибках — не враги разработчика, а его друзья. Они помогают выявить проблему до того, как программа попала к пользователю.

Попробуем ошибиться по-другому, пропустим операцию вывода в поток:
--
cout "Very nice day!"s << endl; 
----

Компилятор, тем не менее, продолжает вторить уже знакомое:
--
/exposed/submission/submission.cpp:6:9: error: expected ‘;’ before string constant
    6 |     cout  "Very nice day!"s << endl;
      |         ^ ~~~~~~~~~~~~~~~~
      |         ; 
----

Компилятор — не оракул. Догадаться, какая именно операция пропущена, он не может. Поэтому предлагает просто завершить команду. Сделайте из этого важный вывод - Не стоит сразу делать то, что советует компилятор. При появлении ошибки нужно вначале разобраться, в чём её причина, и только потом исправлять.

У разработчиков есть друзья помимо сообщений об ошибках. Знакомьтесь — warning. Предупреждения выдаются, когда программа синтаксически корректна и может скомпилироваться, но компилятору что-то кажется подозрительным. В нашем курсе компилятор настроен так, что программы с предупреждениями не принимаются. Аккуратный программист всегда исправляет их. 

Рассмотрим пример:
--
cout ; "Happy coding"s ; endl; 
----

В нём по ошибке написаны ; вместо <<. Такая программа допустима, но она состоит из трёх команд, которые ничего не делают. Это кажется компилятору подозрительным, и он выдаст следующее:
--
/exposed/submission/submission.cpp: In function ‘int main()’:
/exposed/submission/submission.cpp:6:5: error: statement has no effect [-Werror=unused-value]
    6 |     cout ; "Happy coding"s ; endl;
      |     ^~~~
compilation terminated due to -Wfatal-errors.
cc1plus: all warnings being treated as errors 
----

Суть сообщения — statement has no effect. Далее компилятор указывает, что это стало ошибкой, потому что мы попросили считать все предупреждения ошибками.

Конечно, в реальной разработке эту настройку можно отключить. Но тогда вы рискуете пропустить ошибку, и пользователь столкнётся с неправильно работающей программой. Подумайте, стоит ли оно того.

Переменные

Чтобы работать с данными в памяти компьютера, нужно придумать имя, по которому вы будете к ним обращаться. Это имя называют переменной, а данные — её значением:
--
// объявление переменной и задание значения:
string text = "Hello from C++ ^_^"s;

// пример использования переменной:
cout << text << endl; 
----

Ещё один способ вывести то же самое, используя сразу две переменные:
--
string text_begin = "Hello from"s;
string text_end = "C++ ^_^"s;

// выводим обе переменные, не забывая пробел между ними:
cout << text_begin << " "s << text_end << endl; 
----

Слово string, написанное перед именем переменной — это её тип. Он задаётся при первом упоминании, то есть при объявлении переменной и не может меняться. string используется для строковых переменных. Объявление переменной — отдельная команда, после неё нужна точка с запятой.

При использовании переменной тип string не пишется. Достаточно просто имени. Использованная переменная может быть практически в любом месте, например, при выводе, в формуле или для задания другой переменной.

Имя переменной может быть любым, главное, чтобы оно:
- состояло исключительно из латинских букв, цифр или знаков - подчёркивания (_),
- не начиналось с цифры,
- не совпадало со встроенным в язык ключевым словом,
- было уникальным.

Имя, удовлетворяющее перечисленным критериям, называется идентификатором. А вот набор допустимых типов строго определён. В процессе прохождения курса вы узнаете, как создавать свои типы.

В C++ уникальность обязательна только в пределах области видимости. В дальнейшем вы узнаете, как создавать переменные с одинаковыми именами, чтобы они не конфликтовали между собой.

В курсе имена переменных будем писать строчными буквами. Это часть нашего style guide.

Но есть правило ещё важнее — имя переменной придумывайте так, чтобы легко было догадаться, какие данные она хранит. Например, переменная с именем user_name хранит имя пользователя, а с именем user_age — его возраст. К сожалению, компилятор не сможет проверить соблюдение этого правила, поэтому программист должен проявлять ответственность сам.

В этом коде сразу же при создании переменной ей присвоено значение. Со значениями переменных можно выполнять две основных операции — чтение и запись.

При чтении вы просто пишете имя переменной, как это сделано в команде: cout << text << endl;. Читать переменную можно сколько угодно раз, значение при этом не меняется.

Запись нового значения в переменную делается операцией присваивания. Первое присваивание, выполняемое прямо в команде объявления, называется «инициализация».

// Объявляем переменную и сразу инициализируем её строкой:
--
string text = "Hello from C++ ^_^"s;
cout << text << endl;

// Записываем в переменную новое значение, стирая старое.
// Значение переменной text изменяется на строку Happy coding!
text = "Happy coding!"s;
cout << text << endl; 
----

Результат:
--
Hello from C++ ^_^
Happy coding! 
----

Смысл выражения x = y не такой как в математике. Операция присваивания — это команда заменить значение переменной x на значение y.

Чтобы проверить, равны ли два значения, используют операцию ==, но об этом позже.

Если поместить название переменной в кавычки, компилятор посмотрит на её название как на строку, а не как на имя переменной. Следовательно, не будет подставлять значение переменной.

Существуют переменные, для которых доступно только чтение, но не запись. Они называются константными.

Переменные могут использоваться для того, чтобы не повторять одно и то же выражение многократно:
--
string name = "Бенедикт Хант"s;

cout << "- Здравствуйте, меня зовут "s << name << "!"s << endl;
cout << "Посмотрите, пожалуйста, почту на имя "s << name << "."s << endl;
cout << "- На имя "s << name << " имеются три письма!"s << endl; 
----

Результат:
--
- Здравствуйте, меня зовут Бенедикт Хант!
Посмотрите, пожалуйста, почту на имя Бенедикт Хант.
- На имя Бенедикт Хант имеются три письма! 
----

Мы не просто сократили написание, но получили возможность менять имя персонажа, переписывая всего лишь одну строчку кода. Так мы не рискуем допустить ошибку, забыв поменять имя в одном из мест его использования:
--
// Изменена только одна строчка кода:
string name = "Роберт Райт"s;

cout << "- Здравствуйте, меня зовут "s << name << "!"s << endl;
cout << "Посмотрите, пожалуйста, почту на имя "s << name << "."s << endl;
cout << "- На имя "s << name << " имеются три письма!"s << endl; 
----

Результат:
--
- Здравствуйте, меня зовут Роберт Райт!
Посмотрите, пожалуйста, почту на имя Роберт Райт.
- На имя Роберт Райт имеются три письма! 
----

Компьютер в переводе с английского — это вычислитель. Большинство работы происходит с числами. Даже строки — всего лишь абстракция, ведь каждый символ — некоторое число в памяти компьютера. В C++, разумеется, есть средства для работы с числами. Давайте познакомимся с новым типом — int:
--
int answer = 42;
cout << answer << endl; 
----

Результат:
--
42
