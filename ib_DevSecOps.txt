в Linux все пути начинаются с корневой директории [/]

change directory - 
--
cd .

----

если набрать просто cd то перейдешь в домашнюю директорию своего пользователя Linux -

cd

при указании пути, куда требуется перейти, есть 2 варианта - 

- абсолютный путь (лучший вариант, чтобы не было проблем) -

- относительный путь указывается от того места, где ты находишся прямо сейчас

- перейти в домашнюю директорию своего пользователя

ls -l /var/log
----

теперь посмотрим на вывод - 
--
-rw-r--r--   1 root   root   2619 Dec 16 19:07   alternatives.log.1
-rw-r--r--   1 root   root   1834 Oct 11 12:54   alternatives.log.2.gz
drwxr-xr-x   2 root   root   4096 Mar  6 01:28   apt
----

если первым знаком идет минус [-], это означает что это файл 

если первым знаком идет [d], это означает что это директория

если первым знаком идет [l], это означает что это символьная ссылка

если первым знаком идет [s], это означает что это соккет 

кроме этого могут быть и другие файлы - сокеты, блочные устройства и т.д.

в Linux всё является файлом, даже если он специфический, директория это тоже файл, но специальный 

далее - rwx - права доступа - 
r - read (читать)
w - write (писать)
x - execute (запускать) - в Linux файлы можно запускать как скрипты, а для директории означает что можно заходить внутрь этой директории (для файла - это запускать, для директории - это заходить)

далее (первые три знака) написано чьи это права доступа - root (собственник этого файла)
далее - (следующие три знака) root (группа собственников)  
далее - (последние три знака) указаны права доступа для всех остальных  

следующая колонка - размер в байтах 

следующая колонка - время, когда этот файл отредактирован

Чтобы зайти под пользователем root в Ubuntu через WSL -
1 - Если вы вошли под обычным пользователем и хотите переключиться на root, используйте одну из следующих команд:
--
sudo -i
---- 
- Самый простой способ. Система запросит пароль вашего текущего пользователя (созданный при установке Ubuntu), после чего вы получите полный доступ root с загрузкой его окружения
2 - 
--
sudo su - 
---- 
- Аналогичный метод переключения на суперпользователя
3 - 
--
sudo -s
---- 
- Переключает в оболочку root, но сохраняет текущее окружение (переменные и текущую папку)

переключится на root в обычном Linux - 
--
su -
----

--
su 
su -l 
su --login 
----

sudo

cd ~/Documents

еще раз про права доступа - rwx rwx rwx -
- собственник 
- группа собственников 
- все остальные 

чейндж мод (изменить права доступа) - 
--
chmod [<SET><ACTION><PERMISSION>]... FILE 
----

кому мы выдаем (меняем, убираем) права доступа - 
- u - владелец (user)
- g - группа (group)
- o - остальные (others)
- a - все (all)

[+] - добавить права доступа 
[=] - четко указать 
[-] - забрать права доступа

добавить пользователю (user) право на запуск (x) файла hello.sh - 
--
chmod u+x hello.sh
----

поменять владельца на файл или директорию - 
--
chown [OPTIONS] [OWNER] FILE 
----

поменяем владельца на root, потребуется пароль - 
--
sudo chown root hello.sh 
----

если меняем права для root, суперпользователя, привелигированного пользователя - то нужно писать - sudo

cat hello.sh

head [OPTIONS] [FILE]
tail [OPTIONS] [FILE]

head hello.sh

--
tail hello.sh

head -n 5 hello.sh

--
tail -n 12 hello.sh

копирование файлов - cp (copyng files) -
--
cp [OPTIONS] SOURCE DESTINATION
----
- опции необязательны, указывается откуда и куда скопировать 

--
cp /home/w/for_tests.txt /home/w/my_files/
----

копируем файл passwd в то место, где сейчас находимся - 
--
cp /etc/passwd .
----

при выполнении команды cp копируются также и пермишионы (права доступа к файлу или директории)

dd - дейта дубликатор, дубликатор данных, соответственно дублирует данные на низком (битовом) уровне - 
--
dd [OPTIONS] OPERAND 
----

с помощью dd можно - 
- полностью скопировать (склонировать) диск 
- записать загрузочную флешку 
- склонировать CD-ROM
- восстановить загрузчик 
- безопасно удалить данные, чтобы никто не восстановил 
- создать свап-файлы (файлы подкачки)

--
dd if=/dev/zero of=/tmp/swapex bs=1M count=50
----
- if - откуда и что копируем
- of - куда копируем (можно в файл, можно диск в диск)
- bs=1M - размер блока данных 1 мегабайт (варианты - K, M, G, T)
- count=50 - сколько раз повторить, если не знаешь размер диска то его указывать не нужно (ни bs, ни count), а если нужно создать файл конкретного размера, то указывать нужно

mv - moving files - используется для перемещения - 
--
mv SOURCE DESTINATION 
----

при перемещении атрибуты файла (дата создания и права доступа) сохраняются

переместить файл for_tests.txt в дерикторию my_files - 
--
mv /home/w/for_tests.txt /home/w/my_files/
----

можно перемещать сразу несколько файлов - 
--
mv file_1.txt file_2.txt /home/w/my_files

в Linux переименовать файл нельзя, можно только переместить файл под новым именем - 
--
mv /home/w/file_111.txt /home/w/file_222.txt

регулярные выражения -
--
. - это любой один символ 

[ ] - это для одного символа в диапазоне

[^ ] - это отрицание - не этот символ? будет исключатся из потока данных

* - может быть символ прошлый а может и не быть 

^ - совпадение в начале строки 

$ - совпадение на конце строки 

+ - один или больше от прошлого выражения 

? - предидущий шаблон является необязательным, может быть и может не быть

{ } - минимум, максимум, точное совпадение с предидущими выражениями
 
| - логическое или (or)

( ) - используется для создания групп 
----

в одинарных кавычках происходит игнорирование любых спецсимволов -
--
'abc'
----

в двойных кавычках тоже происходит игнорирование спецсимволов, но кроме трех спецсимволов - $, \, ` (доллар, обратный слеш, обратная кавычка) -
--
"abc"
----

все что в обратных кавычках - это вложенная команда, эту команду bash должен выполнить, так можно делать команды внутри команды -
--
`abc`
----

лучше писать всегда в одинарных кавычках -
--
grep 'run' /etc/passwd
----

поиск данных в начале строки - 
--
grep '^root' /etc/passwd
----
- в файле /etc/passwd ищем строку, где 'root' строго в начале строки

найти все строки, оканчивающиеся на букву n - 
--
grep 'n$' /etc/passwd
----

выведем слова, которые начинаются на букву d, потом идут любые два символа, и заканчиваются на букву a - 
--
grep 'd..a' /etc/passwd
----
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
landscape:x:104:105::/var/lib/landscape:/usr/sbin/nologin
------
- по данному паттерну поиска вывелись и результаты из середины других слов

найти любые строки, которые имеют минимум 4 символа - 
--
grep '....' /etc/passwd 
----

если при фильтрации выводится много результатов, то нужно условиями сужать фильтрацию -
--
grep 'r..t' /etc/passwd
----

в [ ] происходит совпадение одного символа со списком или диапазоном, выведем строки в которых есть цифры - 
--
grep '[0-9]' /etc/passwd
----
- выводит все строки, где есть хотя бы одна цифра 

найти строки, в которых есть хоть один нецифровой символ - 
--
grep '[^0-9]' /etc/passwd
----
- не вывелись строки, которые содержат только цифры

если нужно найти строки, содержащие точку - 
--
grep '[.]' /etc/passwd
----

найти все строки, в которых есть буквы r, буква e опционально, и заканчивающиеся на букву d - 
--
grep 're*d' /home/w/Documents/red.txt 
----
red
reeed
rd
reed
------
- знак * перед буквой e говорит о том, что буква e может быть в строке, а может и не быть  

--
grep 'r[oe]*d' /home/w/Documents/red.txt 
----
red
reeed
rd
rod
reed
------
- в скобках [oe] - означает любой из этих двух символов

то есть в [ ] мы указываем список (abc) или диапазон (0-9)

буква e может быть а может не быть в строке - 
--
grep '[e]*' red.txt 
----

чтобы одна e была точно, а вторая e опционально - 
--
grep 'ee*' red.txt 
----

grep может фильтровать не только текст в файлах, но и поток ввода (например с клавиатуры) -
--
grep 'red' 
----
- выведет строку, где есть слово 'red'

прервать ввод grep-а - CTRL + D
