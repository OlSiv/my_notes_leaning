как устроена файловая система linux 

в linux не применяется слово "папка", используются слова "каталог" или "директория"

у команд в linux обычно есть флаги (опции, аргументы) - 
--
ls -l -a
ls -la
----
-l -a - это флаги (опции)

опции не всегда, но часто можно объединять (-l - a, это то же, что и -la)

директории linux - 
- bin (бинарные файлы) - основные исполняемые файлы, команды, которые можно вызвать напрямую из терминала
- sbin - то же, что и в bin, но немного по другому, там то, что используется рутом (root, суперпользователем) например такие - ifconfig, ip a
- usr (юникс систем ресорсес) - системные ресурсы, библиотеки, файловая система в файловой системе
- etc - редактируемые текстовые конфигурации 
- home - домашняя директория пользователей (кроме root)
- tmp - защищает при старте системы, там хранятся временные файлы процессов, программ и т.п. 
- proc - файлы процессов 
- root - директория суперпользователя 
- opt - опциональный (сторонний) софт 
- usr/share - тут тоже сторонний софт для всех пользователей 
- mnt (использовалась в старых версиях, сейчас используется media) - файловая система, диски

даже если в первой команде будет ошибка, то вторая команда все равно выполнится -

создать файл -
touch file_111.txt 

права доступа - rwxrwxrwx 

первые rwx - владелец
вторая rwx - для группы, где состоит владелец
третья rwx - для всх остальных

r - чтение 
w - запись 
x - запуск

chmod 777 file_111.txt 

как тремя цифрами задаются права доступа -
r - 4
w - 2
x - 1
- тут нужно брать сумму от 0 до 7 (для каждой из трех групп пользователей - юзер, группа, все остальные)

- в начале - это файл 
d в начале - это директория 

find - ищет (где (по умолчанию ищет в текущей директории) и что) -
--
find /home/w/Documents file_111.txt 


find /home/w/Documents -name file_111.txt 
----
- name - позволяет искать как в заданной директории так и во всех вложенных в заданную директориях 

можно найти все файлы, которые соответствуют какому-либо паттерну - 
--
find file*.txt
----
- * - заменяет любой знак или любую группу знаков, в данном случае найдет все файлы txt начинающиеся с file (как пример - filename.txt, file123.txt и т.п.)  

найдем все файлы с одним неизвестным символом после file в названии -
--
find file?.txt
----

найдем все файлы с одной строчной буквой [a-z] после file в названии -
--
find file[a-z].txt
----

найдем все файлы со строчными буквами [a-z] после file в названии -
--
find file[a-z]*.txt
----

найдем все файлы с цифрой [0-9] после file в названии -
--
find file[0-9].txt
----

найдем все файлы с цифрами [0-9] после file в названии -
--
find file[0-9]*.txt
----
- тут найдет не только с одной цифрой, но и с несколькими цифрами 

nano file.txt

cat file.txt
