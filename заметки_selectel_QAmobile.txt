Урок 2. Типы мобильных приложений

Рассмотрим три основных типа мобильных приложений: нативные, гибридные и PWA. Начнем с базовых определений, а в конце раздела закрепим новые знания и разберем отличия.

Нативные приложения

Это приложения, которые разработаны для конкретной мобильной операционной системы (ОС) — например, iOS или Android — с использованием языков и инструментов, рекомендованных вендором. Так, актуальный язык программирования для iOS — Swift (ранее был широко распространен Objective-C), для Android — Kotlin или Java.

Ключевые характеристики нативных мобильных приложений — высокая производительность и нативный интерфейс. Под нативным интерфейсом подразумевается соответствие мобильных приложений гайдлайнам — наборам правил, стандартов и рекомендаций.

Гайдлайны создают компании-разработчики операционных систем, например, Apple или Google, чтобы приложения выглядели и работали единообразно и предсказуемо. У Apple это Human Interface Guidelines (HIG) - (https://developer.apple.com/design/human-interface-guidelines), у Google — Material Design - (https://m3.material.io/).

Несоблюдение гайдлайнов может привести к отклонению приложения на этапе проверки в магазине (App Store или Google Play), так что оно не будет выпущено для пользователей до устранения ошибок. Еще одна важная особенность нативных приложений — прямой и полный доступ ко всем API и функциям устройства: камере, GPS, Bluetooth и т. д.



Теперь рассмотрим ключевые преимущества и недостатки нативных приложений.

Преимущества -
- Отличный пользовательский опыт (UI/UX). Внешний вид элементов, жесты, навигация, анимации соответствуют поведению операционной системы и других нативных приложений.
- Есть доступ к аппаратным функциям. Приложение может сразу использовать камеру, геолокацию, микрофон и т. д. При этом после обновления ОС новые функции становятся доступны сразу, в отличие от гибридных приложений, где нужно ждать поддержки со стороны фреймворка.
- Работа в автономном режиме (офлайн).

Недостатки -
- Высокая стоимость и время разработки. Фактически, нужно разрабатывать, тестировать и поддерживать два разных приложения.
- Обновления должны проходить проверку в магазинах приложений. Это занимает дополнительное время.



Гибридные приложения

Понятие «гибридное приложение» может включать два основных значения -

1 - Первый вариант — это нативное приложение, которое использует элемент WebView (компонент для отображения веб-контента). В этом случае весь основной интерфейс и логика работы могут быть реализованы внутри WebView, при этом приложение сохраняет доступ к некоторым функциям смартфона.
Такой подход позволяет команде разработки быстро и экономично переиспользовать логику, которая уже реализована на сайте. Чаще всего метод использует бизнес для оперативной проверки гипотез: если продукт оказывается востребованным, функционал переписывают на «натив». Ключевой фокус в тестировании таких приложений — производительность (загрузка, отзывчивость, потребление памяти), а также корректная работа жестов в WebView.

2 - Второй вариант — это приложения, разработанные с использованием веб-технологий (HTML, CSS, JavaScript) и затем упакованные в нативный контейнер для мобильных платформ. Наиболее популярные фреймворки, которые можно отнести к этому подходу — React Native, Flutter, Kotlin Multiplatform (KMP), Xamarin. Познакомимся с некоторыми из них ближе -

- React Native — кроссплатформенный фреймворк для разработки мобильных и настольных приложений на JavaScript и TypeScript. Поддерживает платформы Android, Android TV, iOS, macOS, tvOS, Web, Windows и UWP.

- Flutter — фреймворк от Google, который позволяет разрабатывать приложения для шести платформ: Android, iOS, Linux, MacOS, Web и Windows. Для разработки используется язык Dart (тоже созданный Google).

- Kotlin Multiplatform (KMP) — технология, которая позволяет использовать язык Kotlin для написания кода под Android, iOS, декстоп- и веб-приложения.

- Xamarin — фреймворк для разработки приложений на C# для iOS, Android, Windows и Mac. Был приобретен компанией Microsoft в 2016 году.

Несколько примеров мобильных приложений: на React Native — Pinterest и Skype, на Flutter — Google Ads и Alibaba, на KMP — Netflix и Philips.
Важно учесть, что каждый из фреймворков реализует отрисовку UI по-своему. Рассмотрим на примерах React Native и Xamarin, чтобы сформировать общее понимание принципов работы.
