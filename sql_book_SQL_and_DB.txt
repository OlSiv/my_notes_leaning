Подключение к базе, первый запрос 

Для доступа к БД требуются следующие реквизиты - 

- адрес сервера - имя или IP-адрес компьютера в сети, на который установлена СУБД, в реальных проектах обычно адрес отличается от адреса текущего компьютера, у нас в учебной настройке будет localhost или IP 127.0.0.1 (если при установке прописали TCP-порт не по умолчанию, при подключении важно указывать его тоже)

- имя для базы данных - чтобы различать базы данных, которых может быть несколько на одном сервере 

- логин и пароль - мы для обучения будем использовать логин и пароль администратора (суперпользователя) - 
- MySQL - root 
- PostgreSQL - postgres 
- MS SQL - sa 
но в реальной работе их использовать не нужно, это угроза безопасности и целостности данных

Чтобы добавить установленную версию PostgreSQL в переменную окружения PATH на Windows, выполните следующие шаги.

### Шаг 1: Определите путь к установленной PostgreSQL

Убедитесь, что вы знаете, где находится установленная версия PostgreSQL. Обычно PostgreSQL устанавливается в каталог, подобный этому:
--
C:\Program Files\PostgreSQL\18\bin
----
Замените 18 на актуальную версию, если она отличается.

### Шаг 2: Откройте настройки переменных среды

1. Нажмите клавишу Win + R, чтобы открыть окно "Выполнить".
2. Введите sysdm.cpl и нажмите Enter. Это откроет свойства системы.
3. Перейдите на вкладку Дополнительно.
4. Нажмите кнопку Переменные среды.

### Шаг 3: Обновите переменную PATH

1. В нижней части окна "Переменные среды" найдите раздел Системные переменные.
2. Найдите переменную Path и выберите её, затем нажмите Изменить.
3. В окне редактирования выберите Создать и вставьте полный путь к каталогу bin PostgreSQL, например:
--
C:\Program Files\PostgreSQL\18\bin
----
4. Нажмите OK, чтобы сохранить изменения.

### Шаг 4: Проверьте изменения

1. Откройте командную строку (нажмите Win + R, введите cmd и нажмите Enter).
2. Введите команду:
--
psql --version
----
Если PostgreSQL добавлен в PATH, вы увидите установленную версию.
